    Getting Started

    What is LAML and why should I use it?

    +

    LAML (Lightning Abstract Markup Language) is a modern, fast scripting language designed for simplicity and performance. It combines the best features of popular languages with a clean, intuitive syntax.

    Key benefits:

    • ‚ö° Lightning-fast execution
    • üß© Simple, readable syntax
    • üîß Powerful built-in functions
    • üåê Cross-platform compatibility
    • üéØ Zero-dependency runtime

    How do I write my first LAML program?

    +

    Here's a simple "Hello World" example:

    // hello.lm
print("Hello, World!")

// Variables and functions
let name = "LAML"
function greet(user) {
    return "Welcome to " + user + "!"
}

print(greet(name))

    Save this as hello.lm and run it with:

    laml hello.lm

    What programming paradigms does LAML support?

    +

    LAML is designed to be multi-paradigm, supporting:

    • Procedural: Traditional function-based programming
    • Functional: First-class functions, closures, and immutability
    • Object-oriented: Classes, inheritance, and encapsulation
    • Declarative: Express what you want, not how to do it

    You can mix and match these approaches as needed for your project.

    The installer says "command not found". What should I do?

    +

    This usually means the LAML binary isn't in your system PATH. Try:

    1. Restart your terminal - The installer should have updated your PATH
    2. Check if LAML was installed:
      which laml
    3. Manually add to PATH:
      export PATH="$HOME/.laml/bin:$PATH"

    If problems persist, check our troubleshooting guide.

    Syntax & Language Features

    How do I declare variables and functions?

    +

    LAML uses simple, clear syntax for declarations:

    // Variables
let message = "Hello"        // Mutable variable
const PI = 3.14159          // Immutable constant
var global_var = 42         // Global variable

// Functions
function add(a, b) {
    return a + b
}

// Arrow functions
let multiply = (x, y) => x * y

// Anonymous functions
let numbers = [1, 2, 3, 4, 5]
let doubled = numbers.map(function(n) {
    return n * 2
})

    Does LAML support object-oriented programming?

    +

    Yes! LAML has full support for classes and objects:

    class Person {
    constructor(name, age) {
        this.name = name
        this.age = age
    }
    
    greet() {
        return "Hi, I'm " + this.name
    }
    
    static species() {
        return "Homo sapiens"
    }
}

// Inheritance
class Developer extends Person {
    constructor(name, age, language) {
        super(name, age)
        this.language = language
    }
    
    code() {
        return this.name + " codes in " + this.language
    }
}

let dev = new Developer("Alice", 25, "LAML")
print(dev.greet())  // "Hi, I'm Alice"
print(dev.code())   // "Alice codes in LAML"

    How do I handle errors and exceptions?

    +

    LAML provides robust error handling:

    // Try-catch blocks
try {
    let result = risky_operation()
    print("Success: " + result)
} catch (error) {
    print("Error: " + error.message)
} finally {
    print("Cleanup code here")
}

// Custom errors
function validate_age(age) {
    if (age < 0) {
        throw new Error("Age cannot be negative")
    }
    return age
}

// Result type pattern
function safe_divide(a, b) {
    if (b === 0) {
        return { ok: false, error: "Division by zero" }
    }
    return { ok: true, value: a / b }
}

    Troubleshooting

    My LAML program runs slowly. How can I optimize it?

    +

    Here are common optimization strategies:

    • Use built-in functions: They're highly optimized 
      // Slow
let sum = 0
for (let i = 0; i < numbers.length; i++) {
    sum += numbers[i]
}

// Fast
let sum = numbers.reduce((a, b) => a + b, 0)
    • Avoid global variables: Use local scope when possible
    • Cache expensive operations: 
      // Cache the length calculation
let len = array.length
for (let i = 0; i < len; i++) {
    // process array[i]
}
    • Use the profiler: laml --profile script.lm

    I'm getting "Syntax Error" but my code looks correct.

    +

    Common syntax issues and solutions:

    • Missing semicolons: While optional, they can prevent ambiguity
    • Unmatched brackets: Check for balanced {}, [], ()
    • Indentation: LAML is flexible, but consistent indentation helps
    • Reserved keywords: Don't use keywords like if, function as variable names

    Use the detailed error mode for better diagnostics:

    laml --verbose script.lm

    How do I debug my LAML programs?

    +

    LAML provides several debugging tools:

    1. Built-in debugger:
      laml --debug script.lm
    2. Debug statements in code: 
      debug("Variable x:", x)
debug.trace()  // Print stack trace
debug.assert(x > 0, "x must be positive")
    3. VS Code integration: Use breakpoints and step-through debugging
    4. Logging: 
      log.info("Processing item:", item)
log.error("Something went wrong:", error)
log.debug("Debug info:", details)

    Advanced Usage

    How do I create and use modules in LAML?

    +

    LAML has a powerful module system:

    // math.lm
export function add(a, b) {
    return a + b
}

export function multiply(a, b) {
    return a * b
}

export const PI = 3.14159

// Default export
export default class Calculator {
    add(a, b) { return a + b }
    subtract(a, b) { return a - b }
}

// main.lm
import { add, multiply, PI } from "./math.lm"
import Calculator from "./math.lm"

let result = add(5, 3)
let calc = new Calculator()
print(calc.add(10, 20))

    You can also import entire modules:

    import * as Math from "./math.lm"
print(Math.add(1, 2))
print(Math.PI)

    Can I call external programs or APIs from LAML?

    +

    Yes! LAML provides several ways to interact with external systems:

    // System commands
let result = system("ls -la")
print(result.stdout)

// HTTP requests
let response = fetch("https://api.example.com/data")
let data = response.json()

// Async/await support
async function getData() {
    try {
        let response = await fetch("https://api.example.com/users")
        let users = await response.json()
        return users
    } catch (error) {
        print("Error fetching data:", error)
    }
}

// File operations
let content = file.read("data.txt")
file.write("output.txt", "Hello, World!")

// Environment variables
let home = env.get("HOME")
env.set("MY_VAR", "value")
