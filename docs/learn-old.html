What is LAML?

LAML (Low Abstraction Machine Language) is a modern compiled programming language that combines the power of low-level system programming with the elegance of contemporary syntax design.

üéØ Core Philosophy

LAML bridges the gap between high-level developer experience and direct hardware control. It's designed for developers who need performance-critical applications but don't want to sacrifice code readability and maintainability.

Your First LAML Program

Let's write and run your first LAML program step by step.

1

Create a .lm file

LAML programs use the .lm extension. Create a file called hello.lm:

~ This is a comment in LAML
bring xcs.class34;

func main() {
    say "Hello from LAML!";
}
2

Run your program

Use the LAML compiler to run your program:

Terminal
$ laml run hello.lm
Hello from LAML!
3

Understanding the code

  • ~ This is a comment - Single-line comments start with ~
  • bring xcs.class34 - Imports the core LAML library
  • func main() - Defines the main function (entry point)
  • say "Hello from LAML!" - Prints text to the console

Variables in LAML

LAML provides three keywords for declaring variables: val, let, and const.

val - Value Variables

Immutable by default, can be reassigned if needed.

val name = "Alice";
val age = 25;
val isActive = true;

let - Mutable Variables

Explicitly mutable variables that can be changed.

let counter = 0;
counter = counter + 1;
say counter; ~ Outputs: 1

const - Constants

Compile-time constants that cannot be changed.

const PI = 3.14159;
const MAX_SIZE = 1024;

üéØ Best Practices

  • Use val by default for most variables
  • Use let only when you need to modify the variable
  • Use const for compile-time constants and configuration values
  • Choose descriptive variable names that explain their purpose

Comments in LAML

LAML provides two types of comments: single-line with ~ and multi-line with {~ ~}.

Single-line Comments

Use ~ for single-line comments:

~ This is a single-line comment
val x = 42; ~ End-of-line comment

Multi-line Comments

Use {~ ~} for multi-line comments:

{~
   This is a multi-line comment
   that can span multiple lines
   and include detailed explanations
~}

func calculate(x, y) {
    {~ 
       Complex calculation logic here
       TODO: Optimize this algorithm
    ~}
    return x * y;
}
